# Архитектура проекта TuxBack

## Общая архитектура

Сервис резервного копирования **TuxBack** реализован как CLI‑утилита с модульной архитектурой. Основная логика приложения разделена на независимые модули, каждый из которых отвечает за отдельную функциональную часть системы.

Пользователь взаимодействует с системой через CLI‑интерфейс. Далее команды маршрутизируются к соответствующим модулям приложения.

Общая схема архитектуры выглядит следующим образом:

```
User
 │
 │ tuxback CLI
 ▼
CLI interface (cli.py / argparse)
 │
 ├── backup.py
 │       создание архивов
 │
 ├── restore.py
 │       восстановление архивов
 │
 ├── storage.py
 │       управление архивами
 │
 ├── scheduler.py
 │       управление расписанием
 │
 └── config.py
         конфигурация и логирование
 │
 ▼
Filesystem storage
 │
 ├── backups/
 ├── schedules.json
 └── tuxback.log
 │
 ▼
systemd timer
 │
 ▼
tuxback run-scheduler
```

CLI‑интерфейс является точкой входа приложения и обеспечивает взаимодействие пользователя с системой резервного копирования.

# Описание модулей

## cli.py

Главный CLI‑интерфейс приложения.

Основные функции:

- обработка аргументов командной строки
- маршрутизация CLI‑команд
- вывод результатов пользователю
- обработка ошибок
- запуск планировщика

Используется стандартная библиотека Python:

```
argparse
```

CLI поддерживает следующие команды:

```
backup
restore
list
delete
status
schedule-add
schedule-list
schedule-delete
run-scheduler
```

---

## config.py

Модуль конфигурации проекта.

Отвечает за:

- основные константы проекта
- путь к директории резервных копий
- путь к файлу расписания
- путь к лог-файлу
- настройку логирования
- версию приложения

Пример конфигурации:

```
APP_NAME = "TuxBack"
APP_VERSION = "1.0.0"
```

Также в модуле реализована функция настройки логирования приложения.

---

## backup.py

Модуль создания резервных копий.

Функции модуля:

- создание архива директории
- генерация уникального имени архива
- сохранение архива в каталог `backups`

Используется стандартный модуль Python:

```
tarfile
```

Архивы создаются в формате:

```
.tar.gz
```

---

## restore.py

Модуль восстановления данных из резервных копий.

Функции:

- поиск архива резервной копии
- извлечение файлов из архива
- восстановление структуры директорий

Результат восстановления записывается в директорию, указанную пользователем.

---

## storage.py

Модуль управления хранилищем резервных копий.

Функции:

- получение списка архивов
- удаление архивов
- проверка существования архива

Данный модуль изолирует работу приложения с файловой системой.

---

## scheduler.py

Модуль планировщика резервного копирования.

Функции:

- добавление задач резервного копирования
- удаление задач
- получение списка задач
- проверка задач, которые необходимо выполнить
- запуск резервного копирования по расписанию

Все задачи сохраняются в файл:

```
schedules.json
```

Каждая задача содержит следующие поля:

```
id
source
interval_minutes
last_run
enabled
```

# Установка приложения

Для установки используется скрипт:

```
install.sh
```

Во время установки выполняются следующие действия:

- копирование файлов проекта в директорию

```
/opt/tuxback
```

- создание глобальной команды

```
/usr/local/bin/tuxback
```

- установка systemd‑сервиса
- установка systemd‑таймера

Удаление выполняется скриптом:

```
uninstall.sh
```

---

# Автоматизация через systemd

Для автоматического выполнения задач используется **systemd‑таймер**.

Во время установки создаются:

```
tuxback-scheduler.service
tuxback-scheduler.timer
```

Таймер запускает команду:

```
tuxback run-scheduler
```

по расписанию (по умолчанию каждую минуту).

Это позволяет автоматически выполнять резервное копирование.

# Docker

Проект содержит **Dockerfile**, позволяющий собрать контейнер с приложением.

Docker используется для:

- контейнеризации приложения
- изолированного запуска
- воспроизводимой среды выполнения

Сборка контейнера выполняется командой:

```
docker build -t tuxback .
```
