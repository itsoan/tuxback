# Архитектура проекта TuxBack

## Общая архитектура

Сервис резервного копирования **TuxBack** реализован как CLI‑утилита с модульной архитектурой. Основная логика разделена на независимые компоненты, каждый из которых отвечает за отдельную функциональную часть системы.

Архитектура проекта выглядит следующим образом:

```
User
 │
 │ tuxback CLI
 ▼
CLI interface (argparse)
 │
 ├── backup.py
 │       создание архивов
 │
 ├── restore.py
 │       восстановление архивов
 │
 ├── storage.py
 │       управление архивами
 │
 ├── scheduler.py
 │       управление расписанием задач
 │
 └── config.py
         конфигурация и логирование
 │
 ▼
Filesystem storage
 │
 ├── backups/
 ├── schedules.json
 └── tuxback.log
 │
 ▼
systemd timer
 │
 ▼
tuxback run-scheduler
```

CLI‑интерфейс является точкой входа приложения и обеспечивает взаимодействие пользователя с системой резервного копирования.

# Описание модулей

## cli.py

Главный CLI‑интерфейс приложения.

Функции:

- обработка аргументов командной строки
- маршрутизация CLI‑команд
- вывод результатов пользователю
- обработка ошибок

Используется библиотека:

```
argparse
```

CLI поддерживает команды:

```
backup
restore
list
delete
status
schedule-add
schedule-list
schedule-delete
run-scheduler
```

---

## config.py

Модуль конфигурации проекта.

Отвечает за:

- основные константы проекта
- путь к директории резервных копий
- путь к файлу расписания
- настройку логирования
- версию приложения

Пример конфигурации:

```
APP_NAME = "TuxBack"
APP_VERSION = "1.0.0"
```

Также в модуле реализована функция настройки логирования.

---

## backup.py

Модуль создания резервных копий.

Функции:

- создание архива директории
- генерация имени архива
- запись архива в каталог `backups`

Используется стандартный модуль Python:

```
tarfile
```

Архивы создаются в формате:

```
.tar.gz
```

---

## restore.py

Модуль восстановления данных.

Функции:

- поиск архива
- извлечение файлов
- восстановление структуры директорий

Результат восстановления записывается в указанную пользователем директорию.

---

## storage.py

Модуль управления архивами.

Функции:

- получение списка архивов
- удаление архивов
- проверка существования архива

Этот модуль изолирует работу с файловой системой.

---

## scheduler.py

Модуль планировщика резервного копирования.

Функции:

- добавление задач
- удаление задач
- получение списка задач
- выполнение задач по расписанию

Все задачи сохраняются в файл:

```
schedules.json
```

Каждая задача содержит:

```
id
source
interval_minutes
last_run
enabled
```

# Автоматизация через systemd

Для автоматического выполнения задач используется systemd‑таймер.

Во время установки создаются:

```
tuxback-scheduler.service
tuxback-scheduler.timer
```

Таймер запускает:

```
tuxback run-scheduler
```

по расписанию (по умолчанию каждую минуту).

Это позволяет выполнять резервное копирование автоматически.

# CI/CD и Docker

В проекте реализованы DevOps‑практики.

## Docker

Docker используется для:

- контейнеризации приложения
- воспроизводимой среды запуска
- изолированного тестирования

Dockerfile позволяет собрать контейнер приложения.

---

## CI/CD

Используется GitHub Actions.

Pipeline автоматически выполняет:

- установку зависимостей
- проверку импортов
- запуск CLI
- проверку логирования
- сборку Docker образа

Это позволяет гарантировать стабильность проекта после каждого коммита.

# Сценарий использования

Ниже приведён пример типичного использования утилиты.

## Установка

```
git clone https://github.com/de-frogg/tuxback
cd tuxback

chmod +x tuxback install.sh uninstall.sh

./install.sh
```

После установки команда доступна глобально:

```
tuxback
```

---

## Создание резервной копии

```
tuxback backup /home/user/data
```

---

## Просмотр архивов

```
tuxback list
```

---

## Восстановление данных

```
tuxback restore backup_2026_03_15.tar.gz ./restore_dir
```

---

## Настройка автоматического резервного копирования

```
tuxback schedule-add /home/user/data 10
```

Это создаёт резервную копию каждые **10 минут**.

---

## Просмотр задач

```
tuxback schedule-list
```

---

## Удаление задачи

```
tuxback schedule-delete 1
```

# Итог

В результате разработки был создан сервис резервного копирования, который:

- работает как CLI‑утилита Linux
- поддерживает автоматическое резервное копирование
- устанавливается как системная программа
- использует systemd для планирования задач
- применяет Docker для контейнеризации
- использует CI/CD для автоматической проверки сборки

Проект демонстрирует практическое применение современных подходов разработки программного обеспечения и DevOps‑инструментов.
